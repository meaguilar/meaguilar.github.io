¿Qué son las Clases, Objetos y Métodos?

Las clases son plantillas para el objetivo de la creación de objetos según un modelo predefinido. Las clases se utilizan para representar entidades o conceptos, cada clase es un modelo que define un conjunto de variables y métodos apropiados para operar con dichos datos.

Los métodos son funciones que sirven para manipular variables dentro de una clase, de ahí viene la primera característica relevante de la programación orientada a objetos que es el encapsulamiento.

El encapsulamiento busca que en lo posible se trate de que sólo pueda accederse a una variable de clase a través de un método de la clase, permitiendo así asignarle valores a las variables( métodos conocidos como setters) y obtener los valores de las variables ( métodos conocidos como getters).

Los constructores de una clase son conocidos como las funciones que crean nuevos objetos, provee la oportunidad de inicializar los objetos al momento de ser creados, asignándoles valores válidos. Una clase puede contener varios constructores permitiendo crear objetos de varias maneras.

Usualmente hacemos la declaración de los métodos al interior de la clase, mientras que la definición se hace por fuera de la clase ayudándose del operador de resolución de ámbito :: para indicar que el método que se está definiendo pertenece a la clase en cuestión.

Consideraciones

Nota:

  • Cada objeto creado a partir de la clase se denomina instancia de la clase.
  • Se pueden declarar múltiples constructores.
  • Las clases implementan los conceptos de encapsulamiento.
  • Si no se desea inicializar los campos en un constructor, se puede definir un constructor por defecto, también llamado constructor noarg.

Métodos

Definir un programa en C + + (Utilizando Programación Orientada a Objetos) que tenga perros, gatos, peces, loros y humanos. Cada uno puede saludar, comer, y moverse.

./classes.hpp

./main.cpp

#include <iostream>
#include "classes.hpp"

using namespace std;

int main (){
    dog a_dog = dog("Chuchi");
    cat a_cat = cat("Misifus");
    fish a_fish = fish("Megalodon");
    parrot a_parrot = parrot("Chayanne");
    human a_human = human("Wilber");

    cout << "Saludo de un perro: "; a_dog.greet();
    cout << "Saludo de un gato: "; a_cat.greet();
    cout << "Saludo de un pez: "; a_fish.greet();
    cout << "Saludo de un loro: "; a_parrot.greet();
    cout << "Saludo de un humano: "; a_human.greet();

    cout << "\nComidas:\n";
    a_dog.eat("Pedigree");
    a_cat.eat("Whiskas");
    a_fish.eat("Goldfish Flakes");
    a_parrot.eat("una galleta");
    a_human.eat("un BigMac");

    cout << "\nMovimiento:\n";
    a_dog.move(300);
    a_cat.move(40);
    a_fish.move(1);
    a_parrot.move(5);
    a_human.move(5000);

}

Ejercicio 2

Un cibercafé planea llevar un control de los juegos que más disfrutan sus clientes, por lo que te contratan a ti para hacer un programa que lleve control de las horas jugadas de cada juego que se ingrese al sistema.

Para realizar esto deberás hacer un programa que implemente programación orientada a objetos, además deberás almacenar todos los juegos que se ingresen al sistema en la estructura dinámica de tu preferencia y que consideres se acopla mejor a las necesidades del sistema.

Diagrama UML a seguir:

El programa debe tener las siguientes funciones:

  1. Agregar nuevos juegos a la estructura dinámica.
  2. Ver todos los juegos almacenados, así como su información (todos sus atributos).
  3. Seleccionar un juego de los almacenados y agregarle x horas de juego
  4. Recorrer todos los juegos almacenados y deberá agregarle una cantidad aleatoria de horas de juego a cada juego.

Notas:

  • Al agregar los juegos deben tener 0 horas jugadas
  • Es importante que no se sobre escriban las horas de juego, sino que agreguen a las que el juego ya tenga en ese momento.
  • Respetar los especificadores de acceso que tiene el diagrama UML, además utilizar sus métodos para llevar a cabo las funciones solicitadas.

Sube los cambios al repositorio y utiliza los comandos en el orden correcto (git add, git commit, git push)

Clona el repositorio con el comando git clone [url]

Nota: Si en dado caso este pop-up se te presenta, elige la opción "Sign in with a code" y sigue los pasos que te indiquen.

Antes de continuar: Asegurate de que no hay credenciales de otros usuarios guardadas en el sistema (Accede al administrador de credenciales de Windows y elimina las que no sean tuyas), que no hay ninguna sesión abierta en el navegador de Chrome y que no hay ninguna cuenta de GitHub con la sesión iniciada(Accede a www.github.com y cierra la sesión si es que hay alguna abierta).

Después de asegurarte que no hay otra sesión, ingresa tus credenciales desde la consola con los siguientes comandos:

git config --global user.name "nombre de usuario en github"

git config --global user.email "email"

Antes de iniciar, crear 1 Rama por Integrante con el formato APELLIDO-NOMBRE-CARNET

Luego al finalizar tu actividad en tu rama:

git add .

git commit -m "Mensaje descriptivo del cambio"

git push

Recuerda: Siempre verifica en el repositorio remoto si los cambios han sido sincronizados.

Haciendo Merge en la rama principal

Cuando realices un push, notarás un cuadro de diálogo de color amarillo, que tiene un botón indicando "Compare & Pull Request". Deberás hacer click en él para comenzar el proceso.

Open a pull request

Básicamente un pull request es una petición para integrar nuestras propuestas o cambios de código a un proyecto. En esta sección, podrás ver un apartado en un cuadro gris, donde nos indica si se puede hacer merge automáticamente, o si existe algún conflicto que tengamos que resolver antes para poder realizar el merge. Aquí, podrás añadir un nombre junto con un comentario de los cambios que has realizado.

Merge pull request

Cuando no existan conflictos, o los conflictos en los archivos ya están resueltos, el botón de "Merge pull request" estará disponible. De no ser así, GitHub nos indicara en qué archivos existen dichos conflictos y nos proporcionará una herramienta de edición de texto en línea para poder arreglarlos.
