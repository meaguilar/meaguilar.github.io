Consejos de Diseño Orientado a Objetos

  • Manténgase cerca del dominio del problema
  • Descubrimiento de objetos frente a invención de objetos
  • Elija sustantivos o frases nominales como clases
  • Los nombres de los métodos deben contener un verbo.
  • Prefijo adjetivos al nombrar clases heredadas
  • No agregue sufijos a los nombres de las clases
  • Evite el mapeo uno a uno del diseño estructurado
  • Reemplace múltiples métodos get-set con operaciones
  • Clases modelo que manejan mensajes como máquinas de estado
  • Utilice const siempre que sea posible
  • Restringir la dependencia de nivel de archivo de encabezado
  • No reinvente la rueda; usar STL
  • La clase con solo métodos de configuración apunta a la delegación perdida
  • Reemplazar una matriz de estructuras con una matriz de objetos
  • Delegar el trabajo a la clase auxiliar
  • Las matrices multidimensionales apuntan a una identificación de clase incompleta
  • Múltiples bucles anidados apuntan a una delegación incompleta
  • Una clase con un gran número de métodos apunta a una identificación de clase incompleta
  • No te excedas con la herencia
  • Preferir la delegación a la herencia
  • No disperses la abstracción
  • Considere un grupo de objetos para dividir el trabajo entre los miembros del equipo
  • Use clases anidadas para clases auxiliares ligeras
  • Utilice plantillas para mejorar la seguridad y el rendimiento de los tipos
  • Divida su código en marco y partes de aplicación
