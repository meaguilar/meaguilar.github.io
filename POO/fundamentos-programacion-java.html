Fundamentos de programación en Java y buenas prácticas

Esta guía introduce los fundamentos de programación en Java, abordando la estructura básica de un programa, tipos de datos, entrada y salida de información, arreglos y estructuras de control. Además, presenta conceptos esenciales como la creación de métodos, el manejo de excepciones y la modularización del código. Finalmente, se incluyen buenas prácticas de desarrollo, convenciones de estilo y documentación con JavaDoc, con el objetivo de fomentar la escritura de código claro, organizado y mantenible.

Estructura Básica de un Programa Java

Un programa Java se compone de clases que contienen métodos. Cada programa debe tener un método main que actúa como punto de entrada de nuestra aplicación.

public class Main {
    public static void main(String[] args) {
        System.out.println("¡Hola Mundo!");
    }
}

En este ejemplo:

  • Main es el nombre de la clase.
  • Por convención se usa PascalCase para nombrar clases.
  • El método main imprime el mensaje “¡Hola Mundo!” en la consola.

Modificadores de acceso

Algunas palabras clave importantes:

  • public
    Permite acceso desde cualquier parte del programa.
    La clase principal Main debe ser pública para ejecutarse.

  • private
    Restringe el acceso solo a la clase donde se declara.

  • static
    Indica que el elemento pertenece a la clase y no a objetos específicos.

🔍 ¿Por qué main es static?
El método main debe ser static porque se ejecuta sin necesidad de crear un objeto de la clase.

Relación archivo / clase

En Java existe una regla fundamental:

✅ El nombre del archivo .java debe coincidir exactamente con el nombre de la clase pública.

Ejemplo

// Archivo: MiPrograma.java

public class MiPrograma {
    // ...
}

Múltiples clases

Si hay varias clases en el archivo:

public class Aplicacion { 
    // ...
}

class Helper {
    // ...
}

El archivo debe llamarse:

Aplicacion.java

porque es la única clase pública.

💡 Consejo

  • Si una clase es public, el archivo debe llamarse igual.
  • Si no hay clases públicas, el archivo puede tener cualquier nombre.
  • Usa siempre PascalCase para clases.

Tipos de Datos y Arreglos

Tipos Primitivos

Java define varios tipos primitivos para almacenar valores básicos.

int age = 25;         // Entero
double price = 19.99; // Decimal
char letter = 'A';    // Carácter
boolean active = true; // Booleano

Tipos de la Biblioteca Estándar

String

La clase String representa cadenas de caracteres.

Es inmutable, lo que significa que su valor no puede modificarse después de crearse.

String day = "Lunes";

Métodos comunes

String texto = "Laboratorio 1";

int longitud = texto.length();
String mayus = texto.toUpperCase();
String sub = texto.substring(0, 4);
String nuevo = texto.replace("Laboratorio", "Practica");

Concatenación

String nombre = "Josse";

String saludo = "Hola, " + nombre + "!";
String concatenado = nombre.concat(" Castillo");

Comparación de Strings

En Java no se recomienda usar == para comparar Strings.

Se debe usar equals().

String nombre1 = "Ana";
String nombre2 = new String("Ana");

System.out.println(nombre1.equals(nombre2)); 
System.out.println(nombre1 == nombre2);
  • equals() compara contenido
  • == compara referencias de objeto

Entrada de Datos con Scanner

La clase Scanner permite leer datos del teclado.

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

        System.out.println("Ingresa tu nombre:");
        String nombre = sc.nextLine();

        System.out.println("Hola, " + nombre);

        sc.close();
    }
}

Leer otros tipos de datos

int edad = sc.nextInt();
double altura = sc.nextDouble();

💡 Nota:
Cierra el Scanner cuando ya no lo necesites.

Arreglos

Los arreglos almacenan múltiples valores del mismo tipo.

Inicialización directa

int[] numbers = {1, 2, 3, 4, 5};

Declaración con tamaño fijo

String[] names = new String[3];

names[0] = "Ana";
names[1] = "Luis";
names[2] = "Pedro";

Estructuras Condicionales

If - Else

Permite ejecutar código dependiendo de una condición.

int note = 85;

if(note >= 90) {
    System.out.println("Excelente");
} else if(note >= 70) {
    System.out.println("Aprobado");
} else {
    System.out.println("Reprobado");
}

Switch

Permite evaluar múltiples casos.

String day = "Lunes";

switch(day) {
    case "Lunes":
        System.out.println("Inicio de semana");
        break;
    default:
        System.out.println("Día no reconocido");
}

Switch moderno (Java 14+)

switch(dia) {
    case "Lunes" -> System.out.println("Inicio de semana");
    case "Viernes" -> System.out.println("¡Fin de semana!");
    default -> System.out.println("Día laboral");
}

Estructuras Repetitivas

For

Se usa cuando sabemos cuántas veces se repetirá el ciclo.

for(int i = 0; i < 5; i++) {
    System.out.println("Iteración: " + i);
}

While

Se usa cuando no sabemos el número exacto de iteraciones.

int counter = 0;

while(counter < 3) {
    System.out.println("Contador: " + counter);
    counter++;
}

Funciones (Métodos)

Los métodos son bloques de código reutilizables.

Método sin parámetros

public static void hello() {
    System.out.println("¡Bienvenido!");
}

public static void main(String[] args) {
    hello();
}

Método con parámetros

public static int sum(int a, int b) {
    return a + b;
}

public static void main(String[] args) {
    int resultado = sum(5, 3);
    System.out.println("La suma es: " + resultado);
}

Manejo de Excepciones

Permite manejar errores sin detener el programa.

try {
   int result = 10 / 0;

} catch(ArithmeticException e) {
    System.out.println("Error: División por cero");

} finally {
    System.out.println("Bloque finally ejecutado");
}

Try With Resources

Permite cerrar recursos automáticamente.

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {

        try (Scanner scanner = new Scanner(System.in)) {

            System.out.print("Ingrese su edad: ");
            int edad = scanner.nextInt();

            System.out.println("Edad ingresada: " + edad);

        } catch (Exception e) {

            System.out.println("Error: Entrada inválida");

        }
    }
}

Buenas Prácticas de Programación

Documentación

Usa comentarios para explicar el propósito del código.

// Este método calcula la suma de dos números
public static int sum(int a, int b) {
    return a + b;
}

JavaDoc

Permite generar documentación automática.

/**
 * Suma dos números enteros.
 * @param a Primer número
 * @param b Segundo número
 * @return Resultado de la suma
 */
public static int sum(int a, int b) {
    return a + b;
}

Modularización

Divide el programa en clases y métodos.

Esto mejora:

  • mantenimiento
  • reutilización
  • organización del código
public class Calculadora {

    public static void main(String[] args) {
        int resultado = sum(10, 20);
        System.out.println("Resultado: " + resultado);
    }

    public static int sum(int a, int b) {
        return a + b;
    }
}

Google Java Style Guide

La Google Java Style Guide define reglas y convenciones para escribir código consistente, legible y mantenible en Java.

Seguir estas reglas permite que diferentes desarrolladores puedan leer, mantener y colaborar en el mismo código fácilmente.

1. Identación

Se utilizan 2 espacios por nivel de indentación.

No se recomienda usar tabulaciones.

Ejemplo

public int calcularTotal(int numeroA, int numeroB) {
  return numeroA + numeroB * 2;
}

2. Longitud de línea

El máximo recomendado es 100 caracteres por línea.

Si una línea es muy larga, debe dividirse para mejorar la legibilidad.

Ejemplo

return "Este es un mensaje muy largo " +
       "que se divide para respetar el límite de longitud de línea";

3. Uso de llaves

Las llaves {} deben colocarse en la misma línea de la declaración.

Ejemplo

if (edad > 18) {
  System.out.println("Es mayor de edad");
} else {
  System.out.println("Es menor de edad");
}

4. Convenciones de Nombres

Las convenciones de nombres ayudan a identificar rápidamente el propósito de cada elemento del código.

4.1 Clases

Formato:

UpperCamelCase

Cada palabra inicia con mayúscula.

Ejemplos

GestorUsuarios
ProcesadorDatos
ServicioPedidos
ControladorUsuarios

Ejemplo en código

public class ProcesadorDatos {

}

4.2 Interfaces

Las interfaces también utilizan UpperCamelCase.

Generalmente representan comportamientos o capacidades.

Ejemplos

ProcesadorPagos
RepositorioUsuarios
ServicioNotificaciones

Ejemplo

public interface ProcesadorPagos {

  void procesarPago(double monto);

}

4.3 Métodos

Formato:

lowerCamelCase

La primera palabra inicia en minúscula.

Ejemplos

calcularTotal()
procesarPedido()
buscarUsuarioPorId()
enviarCorreo()

Ejemplo

public void procesarPedido() {
  contadorPedidos++;
}

4.4 Variables

Las variables también utilizan lowerCamelCase.

Ejemplos

nombreUsuario
precioTotal
contadorPedidos
estadoActivo

Ejemplo

int precioTotal = 100;
String nombreUsuario = "Carlos";

4.5 Constantes

Las constantes se escriben en:

UPPER_SNAKE_CASE

Todas las letras en mayúscula separadas por _.

Ejemplos

MAX_PEDIDOS
TIEMPO_ESPERA
VERSION_API

Ejemplo

public class Configuracion {

  public static final int MAX_PEDIDOS = 100;
  public static final int TIEMPO_ESPERA = 30;

}

4.6 Parámetros de métodos

Los parámetros también usan lowerCamelCase.

Ejemplo

public int calcularTotal(int precio, int cantidad) {
  return precio * cantidad;
}

4.7 Paquetes (Packages)

Los paquetes se escriben completamente en minúsculas.

Formato común:

com.empresa.proyecto.modulo

Ejemplos

com.empresa.usuarios.servicio
com.empresa.pedidos.controlador
com.empresa.productos.repositorio

Ejemplo

package com.empresa.pedidos.servicio;

4.8 Enumeraciones (Enums)

Los nombres de Enums utilizan UpperCamelCase.

Los valores internos usan UPPER_SNAKE_CASE.

Ejemplo

public enum EstadoPedido {
  PENDIENTE,
  PROCESANDO,
  COMPLETADO,
  CANCELADO
}

4.9 Variables booleanas

Se recomienda usar prefijos que indiquen estado.

Prefijos comunes:

es
tiene
puede
debe

Ejemplos

esActivo
tienePermiso
puedeEditar
debeActualizar

Ejemplo

boolean esActivo = true;
boolean tienePermiso = false;

5. Ejemplo completo aplicando las convenciones

package com.empresa.pedidos.servicio;

public class ServicioPedidos {

  private static final int MAX_PEDIDOS = 100;

  private int contadorPedidos;

  public void procesarPedido(String idPedido) {
    if (contadorPedidos < MAX_PEDIDOS) {
      contadorPedidos++;
      System.out.println("Procesando pedido: " + idPedido);
    } else {
      System.out.println("Se alcanzó el número máximo de pedidos permitidos");
    }
  }

  public int obtenerCantidadPedidos() {
    return contadorPedidos;
  }
}

Nota importante

En este documento los ejemplos se presentan en español con fines educativos para facilitar la comprensión de las convenciones.

Sin embargo, en proyectos profesionales se recomienda escribir el código en inglés, incluyendo:

  • nombres de clases
  • métodos
  • variables
  • paquetes
  • comentarios técnicos

Esto se debe a que:

  • El ecosistema de programación está en inglés
  • Facilita la colaboración internacional
  • La mayoría de frameworks, librerías y documentación están en inglés
  • Es el estándar en la industria del software

Ejemplo profesional equivalente:

public class OrderService {

  private static final int MAX_ORDERS = 100;

  private int orderCount;

  public void processOrder(String orderId) {
    if (orderCount < MAX_ORDERS) {
      orderCount++;
      System.out.println("Processing order: " + orderId);
    }
  }

}
